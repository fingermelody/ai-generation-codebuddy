# 需求文档生成规则

## 文档格式

根据用户输入，以及结合你对本地项目工程理解，技术规格，生成的需求文档应包含以下部分：

1. **项目概述**

   - 项目背景
   - 项目目标
   - 目标用户
2. **功能需求**

   - 核心功能列表及模块
   - 功能详细描述
   - 用户流程
3. **非功能需求**

   - 性能要求
   - 安全要求
   - 可用性要求
4. **技术规格**

   - 技术栈
   - 系统架构
   - 第三方集成
5. **优先级及责任矩阵**

   - 优先级
   - 模块Owner
   - 研发处理人
   - 测试处理人
6. **交付物**

   - 里程碑
   - 时间线
   - 验收标准
7. 风险评估

   - 风险项
   - 影响程度
   - 应对策略

## 标准需求生成模板

用户提供信息时，请按照以下结构生成 需求文档：

```markdown

[根据用户输入填写]

[根据用户输入填写]

[详细描述]
```
